# Code of Conduct

## Our commitment

We want tempoll to be a respectful, welcoming project for everyone.

## Expected behavior

- Be respectful and constructive.
- Assume good intent.
- Focus feedback on ideas and implementation, not people.
- Be patient with newcomers.

## Unacceptable behavior

- Harassment, intimidation, or discrimination.
- Personal attacks or abusive language.
- Publishing private information without consent.
- Any other conduct that would be inappropriate in a professional setting.

## Scope

This applies to project spaces (issues, pull requests, discussions if enabled, and other repository communication channels).

## Enforcement

Project maintainers may remove, edit, or reject comments, issues, or contributions that violate this Code of Conduct.

## Reporting

For behavior concerns, open a private maintainer contact request via repository owner channels.

For vulnerability disclosures, use [SECURITY.md](./SECURITY.md) instead of public reports.
